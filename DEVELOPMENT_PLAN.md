# 주식 애널리스트 리포트 모바일 웹앱 — 개발 계획서

> 종목명을 입력하면 기업정보 · 재무제표 · 실적 · 밸류에이션 · 적정주가 · 목표주가를
> 증권사 애널리스트 보고서 수준으로 자동 생성하는 스마트폰 웹앱

---

## 1. 제품 정의

| 항목 | 내용 |
|---|---|
| 입력 | 종목명 또는 종목코드 (자동완성 지원) |
| 출력 | 애널리스트 보고서: 투자의견, 목표주가, 기업개요, 재무분석, 실적 추이, 밸류에이션, 리스크 |
| 대상 시장 | 국내 상장기업 (KOSPI / KOSDAQ) — 1차 범위 |
| 플랫폼 | 모바일 우선 반응형 웹앱 (PWA로 홈화면 설치 가능) |

**핵심 차별점**: 단순 데이터 나열이 아니라 ① 수치 근거가 명시된 밸류에이션(적정주가 산출 로직 공개), ② LLM이 작성하는 자연어 투자 논거(Investment Thesis), ③ 공시 원문(DART) 기반의 팩트 인용.

## 2. 데이터 소스 및 API 연동 전략

### 2.1 DART 전자공시 (OpenDART, opendart.fss.or.kr)
- **발급**: 무료 API 키 (일 20,000건 한도) — Phase 0에서 발급
- **활용 API**:
  - `기업개황(company.json)` — 회사 기본 정보, 업종, 대표자
  - `단일회사 전체 재무제표(fnlttSinglAcntAll)` — 손익계산서/재무상태표/현금흐름표 (연간·분기, 연결·별도)
  - `공시검색(list.json)` — 최근 주요 공시 (실적발표, 유상증자, 자사주 등)
  - `배당에 관한 사항`, `최대주주 현황`, `소액주주 현황` — 주주환원·지배구조 분석
- **주의**: 종목코드 ↔ DART 고유번호(corp_code) 매핑 테이블 필요 (`corpCode.xml` 전체 다운로드 후 DB화)

### 2.2 KRX 정보데이터시스템 (data.krx.co.kr)
- **활용 데이터**: 일별 시세(OHLCV), 시가총액, 상장주식수, 외국인 보유율, 업종(섹터) 분류, 업종별 평균 PER/PBR/배당수익률
- **연동 방식**: KRX Open API (공식) 또는 `pykrx` 라이브러리(비공식 래퍼, 프로토타입용) → 운영 단계에서는 공식 API로 전환
- **용도**: 현재 주가, 밸류에이션 멀티플의 분모(시총), 동종업계(Peer) 비교

### 2.3 보조 데이터 소스
| 소스 | 용도 | 비고 |
|---|---|---|
| 한국투자증권 KIS Developers | 실시간/지연 시세, 호가 | 무료, OAuth 방식 |
| 네이버 검색 API | 최신 뉴스 헤드라인 | 리포트의 "최근 이슈" 섹션 |
| FnGuide/에프앤가이드 컨센서스 | 애널리스트 컨센서스 | 공식 API 없음 — 1차 범위 제외, 자체 추정치로 대체 |

### 2.4 MCP 활용 방법 (기존 MCP 생태계)
- **개발 단계 (Claude Code에서)**:
  - 공개된 **OpenDART MCP 서버** / **KRX MCP 서버**(커뮤니티 구현 다수 존재)를 Claude Code에 등록 → API 응답 스키마 탐색, 샘플 리포트 프로토타이핑, 데이터 검증을 대화형으로 수행
  - 등록 예: `claude mcp add dart -- npx -y dart-mcp-server` (환경변수로 API 키 주입)
- **운영 단계 (앱 백엔드에서)**:
  - 옵션 A(권장): 백엔드가 DART/KRX REST를 직접 호출하고, Claude API에는 **tool use**로 자체 도구를 정의해 전달 — 지연시간·비용 제어 용이
  - 옵션 B: Claude API의 **MCP connector** 기능으로 원격 MCP 서버를 직접 연결 — 도구 코드 중복 없이 재사용
- **원칙**: MCP는 "개발·탐색 가속기"로 쓰고, 운영 데이터 경로는 캐시를 갖춘 자체 데이터 레이어로 고정한다.

## 3. 시스템 아키텍처

```
[모바일 브라우저 (PWA)]
        │ HTTPS
[Next.js 프론트엔드 + API Routes (Vercel)]
        │
        ├── [데이터 레이어] ── DART OpenAPI / KRX API / KIS API
        │         │
        │      [캐시: Upstash Redis]  ← 재무제표(분기 단위), 시세(일 단위) TTL 차등
        │
        ├── [분석 엔진] ── 재무비율 · 밸류에이션 모델 (순수 TypeScript 계산 모듈)
        │
        └── [리포트 생성기] ── Claude API (구조화 출력 + 근거 데이터 주입)
```

- **프론트엔드**: Next.js(App Router) + Tailwind CSS, 차트는 Recharts(재무 차트) + lightweight-charts(주가 차트)
- **백엔드**: Next.js API Routes (서버리스) — API 키는 반드시 서버 측에만 보관
- **LLM**: Claude API — 수치 계산은 코드가, 서술·해석은 LLM이 담당 (수치를 LLM이 창작하지 않도록 분리하는 것이 핵심)
- **DB**: 종목 마스터(코드 매핑)는 SQLite/Postgres, 캐시는 Redis

## 4. 화면 구성 (모바일 우선)

### 4.1 홈 / 검색 화면
- 상단 검색바: 종목명 자동완성 (초성 검색 지원, KRX 상장종목 마스터 기반)
- 최근 조회 종목, 관심종목 리스트
- 시장 요약 (KOSPI/KOSDAQ 지수 미니 위젯)

### 4.2 리포트 화면 — 스와이프 탭 구조
1. **📌 요약 (Summary)** — 첫 화면
   - 투자의견 배지 (매수/중립/매도) + **목표주가** + 현재가 대비 상승여력 %
   - 핵심 투자포인트 3가지 (불릿)
   - 주가 차트 (1M/3M/1Y/3Y 토글)
   - 핵심 지표 카드: PER, PBR, ROE, 배당수익률, 시가총액
2. **🏢 기업개요** — 사업 구조(매출 비중 도넛차트), 대주주/지배구조, 배당 정책
3. **📊 재무분석** — 매출/영업이익/순이익 추이(막대+선 콤보), 수익성(ROE·영업이익률)·성장성(YoY)·안정성(부채비율·유동비율) 지표 테이블, 현금흐름 요약
4. **📈 실적** — 분기별 실적 테이블 및 차트, 전년동기 대비 증감, 최근 실적 관련 공시 요약
5. **💰 밸류에이션** — PER/PBR 히스토리컬 밴드 차트, 업종 Peer 비교 테이블, **적정주가 산출표** (방법론별 결과 + 가중치 + 최종 목표주가), 산출 가정 명시
6. **⚠️ 리스크** — 리스크 요인, 최근 부정적 공시/뉴스

### 4.3 공통 UI
- 하단 고정 바: 현재가 · 등락률 상시 표시
- 리포트 PDF 내보내기 / 링크 공유
- 다크모드 지원
- 하단 면책 문구 고정 (필수 — §7 참고)

## 5. 밸류에이션 방법론 (분석 엔진 스펙)

| 방법 | 산식 | 적용 대상 |
|---|---|---|
| 상대가치 (멀티플) | 적정 PER × 예상 EPS / 업종평균 PBR × BPS | 전 업종 기본 |
| 히스토리컬 밴드 | 과거 3~5년 PER/PBR 밴드 내 위치 | 이익 안정 기업 |
| RIM (잔여이익모델) | BPS + Σ(ROE−요구수익률)×BPS 할인 | 금융·지주 포함 범용 |
| 간이 DCF | FCF 추정 + 영구성장률 | 현금흐름 안정 기업 |

- **목표주가 = 방법론별 결과의 가중평균** (업종 특성에 따라 가중치 자동 조정, 가중치 근거를 리포트에 표기)
- 적자 기업, 금융업, 지주사 등 예외 케이스 분기 처리
- 모든 계산은 코드(단위 테스트 포함)로 수행 — LLM은 결과 해석만 담당

## 6. 단계별 개발 로드맵

### Phase 0 — 준비 (1주)
- [ ] OpenDART API 키 발급, KRX API 신청, (선택) KIS Developers 가입
- [ ] `corpCode.xml` 다운로드 → 종목코드·기업명·corp_code 매핑 마스터 DB 구축
- [ ] Next.js + Tailwind 프로젝트 스캐폴딩, Vercel 배포 파이프라인
- [ ] 개발용 MCP 서버(DART/KRX) Claude Code에 등록 → 데이터 탐색 환경 구성

### Phase 1 — 데이터 레이어 (1~2주)
- [ ] DART 클라이언트 모듈: 재무제표/기업개황/공시검색/배당·주주 API 래퍼 + 응답 정규화(계정과목 표준화)
- [ ] KRX 클라이언트 모듈: 시세, 시총, 업종 분류, 업종 평균 멀티플
- [ ] 캐시 레이어: 재무제표(분기 갱신), 시세(일 갱신), 종목마스터(월 갱신) TTL 차등
- [ ] 통합 도메인 모델: `CompanyProfile`, `FinancialStatements`, `MarketData` 타입 정의
- ✅ 검증: 삼성전자·카카오·신한지주(금융) 3개 종목으로 데이터 정합성 테스트

### Phase 2 — 분석 엔진 (1~2주)
- [ ] 재무비율 계산기 (성장성/수익성/안정성/활동성 20여 개 지표)
- [ ] 밸류에이션 모듈 4종 (멀티플, 밴드, RIM, 간이 DCF) + 목표주가 합성기
- [ ] 업종 Peer 자동 선정 (KRX 업종분류 + 시총 유사도)
- [ ] 단위 테스트: 알려진 종목의 지표를 수작업 계산과 대조
- ✅ 검증: JSON 형태의 "리포트 원자재(raw report data)" 출력 완성

### Phase 3 — LLM 리포트 생성 (1~2주)
- [ ] 프롬프트 설계: 분석 엔진의 수치 데이터 + 최근 공시 요약을 컨텍스트로 주입, "수치는 제공된 데이터만 인용" 제약
- [ ] 구조화 출력: 섹션별 JSON 스키마 (투자의견, 투자포인트, 섹션별 서술, 리스크)
- [ ] 투자의견 결정 로직: 상승여력 기준 규칙(예: +15% 이상 매수) — LLM이 아닌 규칙이 결정
- [ ] 생성 결과 캐싱 (동일 종목·동일 분기 데이터면 재사용)
- ✅ 검증: 실제 증권사 리포트와 나란히 비교 리뷰

### Phase 4 — 프론트엔드 (2~3주)
- [ ] 검색/자동완성, 리포트 탭 화면 6종, 차트 컴포넌트
- [ ] PWA 설정 (manifest, 서비스워커), 다크모드
- [ ] 로딩 UX: 리포트 생성 중 단계별 진행 표시 (데이터 수집 → 분석 → 작성)
- [ ] PDF 내보내기 / 공유
- ✅ 검증: 실기기(iOS Safari, Android Chrome) 테스트

### Phase 5 — 품질 · 운영 (1주 + 지속)
- [ ] E2E 테스트 (Playwright), 오류 모니터링 (Sentry)
- [ ] API 한도 관리 (DART 일 20,000건) — 캐시 적중률 모니터링
- [ ] 면책조항·이용약관 배치, 접근성 점검
- [ ] 배포 및 사용 피드백 반영

**총 예상 기간: 7~11주 (1인 개발 기준)**

## 7. 법적 유의사항 (필수)

- 자동 생성된 목표주가·투자의견은 **자본시장법상 투자자문업 이슈**를 피하기 위해 모든 화면·PDF에 면책 문구 필수:
  > "본 리포트는 공시 데이터 기반 자동 생성 참고자료이며, 투자 권유가 아닙니다. 투자 판단의 책임은 이용자 본인에게 있습니다."
- DART/KRX 데이터 이용약관 준수 (재배포 조건 확인), 시세는 지연 시세임을 명시

## 8. 참고 — 하네스 엔지니어링(Harness Engineering) 핵심 지침

에이전트 기반 코딩(본 프로젝트의 Phase 3 LLM 파이프라인 설계에도 적용)에서 가장 널리 인용되는 지침:

1. **"가능한 가장 단순한 솔루션을 찾고, 필요할 때만 복잡성을 추가하라"** — Anthropic *Building Effective Agents*의 제1원칙이자 하네스 엔지니어링에서 가장 유명한 지침
2. **"하네스의 모든 구성요소는 '모델이 스스로 못 한다'는 가정을 인코딩한다 — 그 가정을 계속 스트레스 테스트하라"** — Anthropic *Harness design for long-running application development*
3. **Generator–Evaluator 루프**: 생성 에이전트와 비판적 평가 에이전트를 분리해 피드백 루프 구성

→ 본 프로젝트 적용: LLM에게 계산을 시키지 않고(가정: 모델의 산술은 신뢰 불가) 코드가 계산·LLM이 서술하는 단순한 단일 패스 구조로 시작하고, 품질이 부족할 때만 리포트 검증 에이전트(Evaluator)를 추가한다.
